## 🏗️ AWS 핵심 개념 정리

### AWS란?
> 인터넷으로 컴퓨터/네트워크 자원을 빌려 쓰는 클라우드 서비스 (Amazon Web Services)

### 🏢 건물 비유로 이해하는 AWS

```
AWS = 거대한 빌딩 전체
 │
 ├─ VPC = 내가 임대한 층 (가상 네트워크 공간)
 │    ├─ 퍼블릭 서브넷 = 외부인 출입 가능한 방 (웹서버)
 │    └─ 프라이빗 서브넷 = 내부인만 출입 가능한 방 (DB)
 │
 ├─ EC2 = 방 안에 있는 컴퓨터 (서버)
 ├─ 보안그룹 = 방 출입문 자물쇠
 ├─ NAT GW = 프라이빗 방에서 외부 인터넷 나가는 통로
 └─ 탄력적 IP = 건물 고정 주소
```

## 🌐 웹 요청 흐름

```
사용자가 브라우저에서 접속
        ↓
인터넷 게이트웨이 (VPC 입구)
        ↓
퍼블릭 서브넷
        ↓
EC2 (웹서버, nginx)
        ↓
프라이빗 서브넷
        ↓
RDS (데이터베이스)
```

## 🛠️ 자주 쓰는 AWS 서비스

| 서비스 | 한 줄 설명 | 비유 |
|--------|-----------|------|
| **EC2** | 가상 서버 컴퓨터 | 빌린 컴퓨터 |
| **S3** | 파일 저장소 | 구글 드라이브 |
| **RDS** | 데이터베이스 | 엑셀 파일 저장소 |
| **VPC** | 가상 네트워크 | 내 전용 네트워크 공간 |
| **IAM** | 사용자/권한 관리 | 출입증 발급 |
| **Lambda** | 서버 없이 코드 실행 | 필요할 때만 켜지는 코드 |

## 🔐 IAM (Identity and Access Management)

> AWS에서 **누가 뭘 할 수 있는지** 권한을 관리하는 시스템

```
사장 (Root 계정)  → 모든 권한
직원 A (IAM 유저) → EC2만 접근 가능
직원 B (IAM 유저) → S3만 접근 가능
인턴   (IAM 유저) → 읽기만 가능, 삭제 불가
```

| 개념 | 설명 |
|------|------|
| **Root** | 계정 생성 시 자동 생성, 모든 권한 보유 (평소엔 사용 X) |
| **IAM 유저** | 실제 작업에 사용하는 계정, 필요한 권한만 부여 |
| **그룹** | 유저들을 묶어 권한 일괄 관리 |
| **역할(Role)** | 서비스에게 권한 부여 (예: EC2가 S3 접근) |
| **정책(Policy)** | 실제 권한 내용이 담긴 문서 |
